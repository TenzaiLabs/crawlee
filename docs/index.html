Tenzai Crawler

Authenticated crawling for operator-led application reconnaissance.

A single-job FastAPI service that runs pre-crawl authentication when needed, sends Katana through Proxify, and turns captured traffic into a sitemap your team can inspect or feed into the next testing step.

Read the docs Start locally

Quickstart

Install dependencies, run the API, and create a crawl job. The service drains one job at a time because the proxy binds a fixed local port. 

# Install runtime dependencies
uv sync --extra test

# Run the API
uv run tenzai-crawler-server

# Create a crawl job
uv run tenzai-crawler create https://example.com

# Check status and read the completed sitemap
uv run tenzai-crawler status <job_id>

How it works

The orchestrator owns job state. Authentication is optional, crawling stays auth-agnostic, and logs are parsed only after the crawl finishes.

POST /jobs
Operator submits a target URL, scope options, and optional auth config.
Auth, if needed
Credentials or a login URL trigger the Playwright-driven auth agent.
Katana crawl
Katana receives headers, seed URLs, scope limits, and exclusion patterns.
Sitemap output
Proxify and Katana logs become a normalized job result.

What it is for

Tenzai Crawler is focused on repeatable crawl capture for trusted operators, not broad public exposure.

Authenticated surfaces

Use manual headers when the operator already has a session, or run the auth agent when credentials or a login URL are supplied.

Safety-aware crawling

Scope rules and blocked URL patterns keep destructive flows, logout routes, and unrelated domains out of the crawl path.

Durable evidence

Job state lands in SQLite, request logs stay on disk, and completed jobs expose a deduplicated sitemap through the API.
